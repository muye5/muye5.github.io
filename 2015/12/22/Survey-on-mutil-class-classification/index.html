1. Multiclass Classification Methods

  • Dataset: $$\lbrace (x_i, y_i) | x_i \in R^n, y_i \in \lbrace 1,…,K \rbrace \rbrace$$

  • Algorithms:

    • Some can be naturally extended from binary to multiclass classification

      1. Neural Networks
      2. Decision Trees
      3. KNN
      4. Naive Bayes
      5. Softmax
      6. SVM

    • Some need special formulations

      1. convert multiclass problem into a set of binary classification
      2. try to pose a hierarchy on the output space

2. Extensible Algorithms

  • Neural Networks

    • Having N binary neurons in output layer
    • One-Per-Class Coding
    • Distributed Output Coding
      1. Each class is assigned a unique binary codeword from 0 to $2^N - 1$
      2. For testing, nearest codeword is considered the winning class according to some distance measure. e.g. hamming distance

  • Decision Trees
    splitting based on some measures
    each leaf node corresponds to a class label

  • KNN
    distance measure

  • Naive Bayes
    maximun a posterior (map)
    $C = argmax_{c} P(C=c | x_1, …, x_d)$

  • Softmax
    e.g. me, crf
    $P(y=k|x) = \frac {e^{w_{k}^{T} \cdot x}} {\sum_{j} e^{w_{j}^{T} \cdot x}}$

  • SVM
    Binary: maximizing the minimum distance from the separating hyperplane to the nearest example
    Multiclass:
    $$min_{w_1,…,w_K} \frac{1}{2}||(w_1,…,w_K)||^2 + C\sum_{ik}\epsilon_{ik}$$
    $$s.t. \forall (i, k), w_{y^i}^{T}x^{i} - w_{k}^{T}x^{i} \ge 1\lbrace y^{i} \neq k\rbrace - \epsilon_{ik}$$
    Conclusion: none of these approaches were superior to using a set of binary SVMs in an ‘All Pairs’ framework

3. Decomposing Into Binary Classification

  • One-vs-All (OVA)
    reduce classifying among K classes into K binary problem
    positive: given class
    negative: other K - 1 classes
    output: maximum output is considered the winner
    constrain: binary classifier should be tuned well

  • All-vs-All (AVA)
    reduce classifying among K classes into K(K-1)/2 binary problem
    positive/negative: pairwise
    output: class with maximum number of votes wins
    advantage: better than one-vs-all in general

  • Error-Correcting Output-Coding (ECOC)

    • training N binary classifier to distinguish between K classes
    • each row of M corresponds to a certain class
    • comparing between output and the given K codeword, which with the minimum (hamming distance) wins
    • $d(M(k), f(x)) = \sum_{i=1}^{N} L(M(k, i), f_i (x))$
    1. Generalized Coding
      from {0, 1} to {-1, 0, +1}
      special-case: OVA and AVA
    2. What is a good matrix M
      codewords are well-separated in hamming distance
      a large hamming distance between pairs of columns and also between each column and the complements of the rest
    3. How to generate the matrix M
      randomly generating 10000 matrices and choosing the one with the highest minimum hamming distance among its rows.
      best coding strategy is problem dependent
    4. Conclusion
      OVA is generally inferior to the other approaches, while no one approach generally outperforms the others

  • Binary Hierarchical Classification (BHC)

    • BHC decomposes a K class problem into K-1 binary meta-class problems
    • The grouping of the individual classes into meta-classes is determined by the class distributions

    • Classes that are similar to each other in the input feature space are lumped into the same meta-class higher up in the tree

    • Fisher’s Discriminant (FD / LDA)

    1. FD is closely related to PCA in that they both look for linear combinations of variables which best explain the data.
    2. FD explicitly attempts to model the difference between the classes of data.
    3. PCA on the other hand does not take into account any difference in classes.
    4. Maximum of FD
      \begin{align}
      W & = P(\omega_{\alpha}) \Sigma_{\alpha} + P(\omega_{\beta}) \Sigma_{\beta} \\
      B & = (\mu_\alpha - \mu_\beta)(\mu_\alpha - \mu_\beta)^T \\
      \hat w & = argmax_w \frac {w^TBw}{w^TWw} \\
      & = W^{-1} (\mu_\alpha - \mu_\beta) \qquad (4)
      \end{align}
    • Split K classes into 2 parts by FD through Annealing
    1. Initialise $P(\Omega_\alpha | \omega_1) = 1$ for some $\omega_1 \in \Omega$ and $P(\Omega_\alpha | \omega) = 0.5, \forall \omega \in \Omega - \omega_1$. Temperature $T = T_0$ (user defined parameter).
    2. Compute the means and covariances of the meta-classes:
      $$\mu_\gamma = \sum_{\omega \in \Omega} P(\omega | \Omega_{\gamma}) \mu_{\omega}, \gamma \in \lbrace \alpha, \beta \rbrace \qquad (6)$$
      $$\Sigma_\gamma = \sum_{\omega \in \Omega} \frac{P(\omega | \Omega_\gamma)}{N_\omega} [\sum_{x \in \chi_\omega} (x - \mu_\gamma)(x - \mu_\gamma)^T], \gamma \in \lbrace \alpha, \beta \rbrace \qquad (7)$$
      $$P(\omega | \Omega_\gamma) = \frac {P(\omega)P(\Omega_\gamma|\omega)}{P(\Omega_\gamma)}, \gamma \in \lbrace \alpha, \beta \rbrace$$
      $$P(\Omega_\gamma) = \sum_{\omega \in \Omega} P(\Omega_\gamma | \omega)P(\omega | \Omega)$$
      $$P(\omega|\Omega) = \frac {P(\omega)}{P(\Omega)} = \frac{P(\omega)}{\sum_{\rho \in \Omega}P(\rho)}$$
      $$P(\omega) = \frac {N_\omega}{\sum_{\omega \in \Omega}N_\omega}$$
    3. Compute the Fisher projection vector w using Eq.(4) with the sample mean and covariance of the meta-class for the parameters in Eqs (6) and (7)
    4. Compute the mean log-likelihood of meta-classes $\Omega_\gamma (\gamma \in \lbrace \alpha, \beta \rbrace)$:
      $$L(\Omega_\gamma | \omega) = \frac {1}{N_\omega} \sum_{x \in \chi_\omega} log p(w^Tx|\Omega_\gamma) \qquad (8)$$
      where the pdf of each $\Omega_{\gamma}$ in the one dimensional projected space is modelled as Gaussian:
      $$p(w^Tx|\Omega_\gamma) = N(w^Tx; w^T\mu_\gamma, w^T\Sigma_{\gamma}w)$$
    5. Update the meta-class posteriors:
      $$P(\omega_\alpha|\omega) = \frac {exp(L(\Omega_\alpha|\omega)/T)}{exp(L(\Omega_\alpha|\omega)/T) + exp(L(\Omega_\beta|\omega)/T)}$$
    6. Repeat Steps 2 through 4 until the incremental increase in Fisher Discriminant is insignificant(e.g. less than 5%)
    7. Compute the entropy of meta-class posterior:
      $$H = -\frac {1}{|\Omega|} \sum_{\omega \in \Omega} [P(\Omega_\alpha|\omega)log_{2}P(\Omega_{\alpha}\omega) + P(\Omega_\beta|\omega)log_{2}P(\Omega_{\beta}\omega)]$$
    8. if $H \lt \Theta_H$ (user defined threshold) stop, otherwise:
      cool temperature: $T = T\Theta_T$, ($\Theta_T \lt 1$ is a user defined cooling parameter)
      go to step 2

  • ECOC vs BHC

    1. ECOC is an ensemble solution
    2. ECOC partitions the input space
    3. ECOC does not take into consideration the affinities among the individual classes. Some of the groupings might result in complex decision boundaries

    4. Though multiple solutions have been proposed, none of them are guaranteed to produce the optimal matrix for a problem at hand

    5. BHC partitions the output space

    6. BHC uses far lesser number of classifiers in most cases
    7. The binary trees produced by the BHC are consistent with those a human expert might have constructed when given just the class label
    8. BHC often provides a valuable tool for extracting domain knowledge and achieve better results when coarser granularity of the output space is acceptable
